# retail_sales-SQL

# 🛒 Retail Sales SQL Analysis

A comprehensive SQL project that analyzes retail sales data using **PostgreSQL**. This project demonstrates SQL skills ranging from basic queries to advanced business analysis using **Aggregate Functions, GROUP BY, HAVING, Subqueries, and Common Table Expressions (CTEs)**.

---

## 📌 Project Objectives

- Analyze retail sales performance.
- Practice SQL from beginner to intermediate level.
- Solve real-world business problems using SQL.
- Build a portfolio-ready SQL project for interviews.

---

## 🛠️ Technologies Used

- PostgreSQL
- SQL
- Git
- GitHub

---

## 📚 SQL Concepts Covered

- SELECT
- WHERE
- ORDER BY
- LIMIT
- Aggregate Functions (SUM, AVG, COUNT, MAX, MIN)
- GROUP BY
- HAVING
- Date & Time Functions
- Business Analysis Queries
- Scalar Subqueries
- Common Table Expressions (CTEs)

---

## 📊 Business Questions Solved

### Basic Queries
- Retrieve all sales records.
- Filter sales by category and customer.
- Sort and limit results.

### Aggregate Analysis
- Total sales revenue.
- Average sales amount.
- Highest and lowest sales.
- Total number of transactions.

### Group By Analysis
- Revenue by category.
- Average sales by gender.
- Number of transactions by category.
- Highest sale in each category.

### Business Analysis
- Top customers by spending.
- Most profitable category.
- Revenue contribution by category.
- Monthly sales analysis.
- Peak sales hours.
- Customer purchase analysis.

### Subqueries
- Transactions above average sales.
- Categories with above-average revenue.
- Customers spending above average.

### Common Table Expressions (CTEs)
- Customer spending analysis.
- Top customers by revenue.
